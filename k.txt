Functional requirements (core features)

User management
Student and staff registration, profile edit, password reset.
Role-based access: Admin, Librarian, Student.
Authentication & authorization
Secure login (username/password) and session management.
Role-based permissions for actions (issue, return, manage inventory).
Catalog & search
Browse/catalogue with metadata (title, author, ISBN, edition, copies, location).
Advanced search (title, author, ISBN, subject, availability) and filters.
Lending operations
Issue book to user with due date; record checkout details.
Return processing with condition notes.
Loan limits and configurable loan periods by role or item type.
Reservations / holds
Place/cancel holds; notify user when book available; FIFO queue.
Fines & payments
Calculate overdue fines; record payments and adjustments.
Inventory & acquisitions
Add/update/remove book records, track copy counts, barcodes.
Purchase order / acquisition tracking (optional).
Notifications
Email/SMS/in-app notifications for due reminders, holds, fines.
Reports & dashboards
Circulation reports, overdue lists, inventory, user activity, acquisition reports.
Import/export & integration
Bulk import/export (CSV/Excel), integration with student database or LDAP.
Audit & logging
Log user actions (issue/return/update) for accountability.
Admin configuration
Configure loan rules, fine rates, working hours, notification templates.
